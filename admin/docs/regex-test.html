Regular Expression Tester

Introduction

Regular expressions are used for blacklist items and to verify that your regular expression is working as expected, ToplistX includes an interface where you can test your regular expressions against specific values. This document provides an overview of that interface.

Accessing the Test Interface

To access the test interface click on the icon in the blacklist interface. This will popup a window where you will be able to enter the regular expression to test and the value to test it against.

Testing Regular Expressions

In the test interface enter the regular expression to test in the field labeled Regular Expression and the value to test it against in the Test String field. Press the Test Regex button to test your regular expression. When you press this button, a request will be sent to your server, so allow a few seconds for it to complete. When the test is completed the results will display in the Results section. The results will show you if the string you entered matched the regular expression, and if so it will show you the part that matched.

In ToplistX the Perl Compatible Regular Expressions format is used for regular expressions. If you are not familiar with regular expressions, please see the Pattern Syntax portion of the PHP manual for additional information.

When entering your regular expression, you do not need to use any delimiters at the beginning and end of the regular expression. For example, you could use this value to test if a string contains an IP address:

\d+\.\d+\.\d+\.\d+

If you then enter "My IP address is 192.168.1.15" and press the button, this will indicate a successful match on "192.168.1.15".
